# Changelog

Все значимые изменения в проекте EskTech SSO будут документироваться в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и проект придерживается [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.0-alpha] - 2026-05-10

### Добавлено
- Полная поддержка OpenID Connect (OIDC) с discovery endpoint
- Подпись JWT алгоритмом RS256 с генерацией RSA ключей
- Интеграция с GitLab CE 17.8.5 (успешно протестирована)
- Docker Compose среда для быстрого тестирования (`docker-compose-gitlab.yml`)
- Расширенный health endpoint с мониторингом всех сервисов
- Redis пул соединений (`RedisPoolManager`)
- Менеджер JWT токенов в Redis (`RedisJWTManager`)
- Поддержка nonce для защиты от replay-атак
- JWKS endpoint для распространения публичных ключей
- UserInfo endpoint для получения данных профиля
- Скрипт автоматической настройки GitLab (`connect-gitlab-sso.sh`)

### Изменено
- Миграция с HS256 на RS256 для JWT подписи
- Изменен статус редиректа с 307 на 303 после логина
- Расширена структура ответа health endpoint

### Исправлено
- Ошибка GitLab "connection to :80"
- Ошибка несоответствия nonce в ID токене
- Ошибка "Method Not Allowed" при аутентификации
- Проблемы с пулом соединений Redis

### Безопасность
- Валидация nonce для предотвращения replay-атак
- Поддержка отзыва токенов (RFC 7009)
- Безопасное хранение приватных ключей (права 600)

## [0.1.0-alpha] - 2026-05-10
### 📦 Первая альфа-версия

**Основные возможности:**
- 🔐 OIDC-совместимый сервер авторизации
- 🔑 Поддержка RS256 и JWKS
- 🐙 Интеграция с GitLab CE
- 📊 Расширенный мониторинг здоровья
- 💾 Redis для управления токенами

**Известные ограничения:**
- Только HTTP (без HTTPS)
- Cookie-based сессии
- Нет rate limiting

**Следующий релиз (0.2-alpha):**
- Поддержка SAML 2.0 для интеграции с Jira
