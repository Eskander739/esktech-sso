# Версия 0.1 Alpha - Поддержка OpenID Connect и интеграция с GitLab

**Дата выпуска:** 10 мая 2026 г.  
**Статус:** Альфа-версия (протестировано и подтверждено)

## 🎯 Обзор

Этот выпуск является важной вехой для EskTech SSO благодаря внедрению полной поддержки **OpenID Connect (OIDC)** и успешной интеграции с **GitLab**. Сервер SSO теперь работает как полностью совместимый OIDC-провайдер, обеспечивая бесшовный единый вход для GitLab и других OIDC-совместимых приложений.

## 🚀 Основные возможности

### 1. Полная реализация OpenID Connect (OIDC)
- Discovery endpoint в соответствии со стандартом (`/.well-known/openid-configuration`)
- Authorization code flow с поддержкой PKCE
- Endpoints для интроспекции и отзыва токенов
- UserInfo endpoint для получения данных профиля пользователя
- JWKS (JSON Web Key Set) endpoint для распространения публичных ключей

### 2. Подпись JWT алгоритмом RS256
- Миграция с HS256 на **RS256** (RSA SHA-256)
- Безопасная генерация пары приватный/публичный ключи
- Автоматическая загрузка ключей из нескольких источников:
  - `/app/keys/private.pem` и `/app/keys/public.pem`
  - Переменные окружения (`SSO_PRIVATE_KEY_PATH`, `SSO_PUBLIC_KEY_PATH`)
  - Пользовательские пути из конфигурации
- JWKS в формате, совместимом с GitLab, с правильным заголовком `kid`

### 3. Интеграция с GitLab
- **Успешно протестировано и подтверждено** с GitLab CE 17.8.5
- Полная конфигурация OIDC-провайдера для GitLab
- Автоматическое создание OAuth-клиента при настройке GitLab
- Поддержка как автоматического, так и ручного указания endpoints
- Соответствие строгим требованиям GitLab:
  - Алгоритм подписи RS256
  - Валидация nonce
  - Корректное сопоставление redirect URI

### 4. Docker Compose среда для тестирования
- Новый файл `docker-compose-gitlab.yml` для быстрого тестирования
- Предварительно настроенные сервисы:
  - PostgreSQL для хранения пользователей и OAuth-данных
  - Redis для управления токенами
  - GitLab CE с автоматической настройкой OIDC
  - Приложение SSO с генерацией ключей
- Автоматические health checks и зависимости между сервисами
- Персистентность данных через volumes

## 🔧 Улучшения

### Расширенный мониторинг здоровья
- Доработан endpoint `/health` с детальным отображением статусов:
  - Проверка подключения к PostgreSQL
  - Проверка доступности Redis
  - Мониторинг каждого OAuth-клиента
  - Отслеживание времени ответа для каждого сервиса
  - Временные метки последней проверки
- Агрегированный статус `overall_status`
- Красивый вывод JSON с правильным форматированием на русском языке

### Интеграция Redis
- Реализован `RedisPoolManager` для пула соединений
- Создан `RedisJWTManager` для управления жизненным циклом токенов:
  - Хранение токенов с привязкой к email пользователя
  - Инвалидация токенов (одиночная или массовая)
  - Индексация токенов пользователя для быстрого поиска
  - Кэширование запросов
  - Утилиты для очистки просроченных токенов
- Проверка соединения с поддержкой таймаута

### Поддержка nonce в OIDC
- Полная реализация валидации параметра state
- Генерация и проверка nonce для защиты от replay-атак
- Корректная передача nonce через все этапы:
  - Authorization request → Сессия → База данных → ID Token
- Исправление ошибки GitLab "nonce does not match"

### Исправление управления сессиями
- Изменен статус редиректа с **307** на **303** после логина
- Решена проблема конвертации POST → GET
- Устранена ошибка "Method Not Allowed" при аутентификации

## 🐛 Исправленные ошибки

| Ошибка | Решение                                                   |
|--------|-----------------------------------------------------------|
| Ошибки SSL при подключении по HTTP | Добавлена поддержка HTTPS                                 |
| Ошибка GitLab "connection to :80" | Корректная конфигурация endpoints с явным указанием схемы |
| Ошибка несоответствия nonce | Полная передача nonce через все этапы OIDC                |
| Method Not Allowed на authorize | Изменен редирект с 307 на 303                             |
| Отсутствие JWKS endpoint | Реализован экспорт RSA-ключей в формате JWKS              |
| Проблемы с пулом соединений Redis | Реализован асинхронный пул с контекстными менеджерами     |
